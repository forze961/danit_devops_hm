**Поясніть роль і значення кожного з наступних компонентів в архітектурі ОС Linux:**

- a) Kernel
- b) Libraries
- c) System utilities

---
## Kernel
Безпосередньо ядро операційної системи. Має низькорівневі інструкції комунікації саме з апаратною складовою пристрою.
Як складові вміщує в собі функціонал, для роботи з зовнішніми пристроями (драйвера), реалізує управління пам*ятю, процессором.

## Libraries
Набір програмних інструкцій (коду) більш високого рівня зібраних за певною класифікацію або категорією (робота з файлами, робота з кодеками, робота з графікою). Комунікують з ядром системи. Містять вихідні інтерфейси для подальшого використання програмами, скриптами, утилітами. 
Існують системні та користувацьки. (системні постачаються разом з дистрибутивом ОС). Файли бібліотек мають розширення SO. Як правило проектуються, програмуються та компілюються таким чином, щоб гарантувати  уніфіковане використання тої чи іншої функції нашою програмою не зважачи на версію ядра або системи. 
Самі бібліотеки також можуть викликали інші бібліотеки, якщо функціонал певної задачі задіює інші категорії бібліотек.
Наприклад: я викликаю js код з під nodejs та використовую бібліотеку ffmpeg для транскодування відео. Я передаю набір команд з параметрами безпосередньо ffmpeg бібліотеці, а вона вже каскодом викликає інші (libavcodec, libavfilter), тому що транскодування передбачає декілька кроків.

## System utilities
Це вже програми, які використовуються кінцевими користувачами для взаємодії з ОС. Вони викликають певний код (функції) саме з бібліотек, про які я описав вище. Мають повністю людино-читаємий синтаксис, що дозволяє швидко освоювати взаємодію з ними.
Системними утилітами можна назвати програми командного рядка, такі як: ls, ps, chmod, etc.
